从零搭建前端开发体系

发表于 2015-10-28   |  

三个月前换了工作，刚接手新公司的代码时，瞬间有种从天堂到地狱的感觉，目录结构混乱不堪，代码质量堪忧，中间夹杂着大大小小的坑，上线的代码也是没用经过仔细优化，在此基础上开发简直如履薄冰，痛苦不堪，刚开发完一个需求，处女座的我就已经完全受不了这种开发模式，于是立马着手建立一套完善的前端开发体系。

技术选型

因为是基于 H5 的前后端分离项目，主要用于移动端，因此着眼点主要有几个方面。

  1. 很好的适配 Android 和 IOS 中的 Webview，兼容各主流移动浏览器包括微信，同时 PC 上也可供查看使用。
  2. 需要提高开发效率，复用公用代码，做到前端组件化，模块化。
  3. 能够快速部署测试环境，快速上线。

下面就逐个介绍下使用的一些解决方案。
构建工具选择了 gulp，关于 gulp 的使用网上已经有大量教程，之前也写了点使用心得。当然随着前端的发展，Webpack 这种提倡纯模块化的构建工具也是越来越火，后面也会逐渐把其加入到 Tool Chains 中。

HTML 方面使用了模板语言 Jade，主要用于 HMLT代码片段的复用，之前的项目使用了 nginx 的 SSI 来复用部分 HTML 代码，但 SSI 过于简单，性能也有所损耗。

CSS 使用了基于 SASS 语法的 Compass 库，Compass 提供了一些常用的 mixin 方便我们书写 CSS，不过目前来看 Compass 在进行构建时，速度略慢，后面可能考虑使用 node-sass 或直接用 Sass 加上 post-css 并使用 Webpack 进行 CSS 的模块构建。

部署方案采用了上家公司使用的 CI 系统 Jenkins 作自动化构建发布方案，配合 gitlab 的 Webhook 让开发人员只需要提交代码，后续的构建及发布流程完全自动化。

移动开发相关

之前一直在做 PC 项目，虽然流量不是很大，但是项目完全的前后端分离，架构也比较复杂，前端部分有大量的逻辑处理，从前端路由到 MVC，再到构建，这种 PC 上的 SPA 应用，更注重用户交互及体验。
而移动端的 HTML5 虽然也采用的是前后端分离的方案，但目前只做简单数据的渲染，并没有太多的数据交互，因而架构相对简单的多，其主要的关注点在于多屏幕的适配，性能的优化及如何与客户端 App 进行交互上面。
多屏幕的适配其实之前也做过一些使用 Media Query 进行响应式开发的方式，单实际开发中遇到不少问题，首先是设计师在设计时需要考虑不同尺寸的手机屏幕，其次是在各种屏幕分辨率下要做不少 hack 来 fix 不同的设备 dpr 带来的问题（比如如何显示物理上的 1px ）。正好遇到手淘的 Flexible 方案，实践了一下发现能解决上面的两个问题，同时手淘自己也在用这套方案，便以此做为目前 H5 的适配上的解决方案，实际使用中又踩了不少坑。

这里也是感谢老东家和前端 leader 张哥，在为知笔记快两年里，学到了非常多得东西，从浏览器插件开发到用 MVVM 框架去开发复杂 SPA 单页面应用，从学着用 Less，Jade 等到去优化构建流程中的每一步，从兼容各种浏览器到玩耍 CSS3、ES6。毕业后就能有机会和一帮对技术极为追求的同事去尝试去学习，简直是一种幸福！
